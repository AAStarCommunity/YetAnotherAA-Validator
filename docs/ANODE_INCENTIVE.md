# aNode 激励与分润模型 —— PGL 的一个具体执行实例

> 本文回答两个问题:**(1) 一个普通个体凭什么愿意帮网络跑一个去中心化节点(KMS + DVT)?**
> **(2) 从纯经济上算,他要花多少、多久回本、月入多少才划算?**
>
> 关联:issue #122(背景与决策清单)· Mycelium PGL(收益分配/不发币红线)· SuperPaymaster v5(角色/信用/SBT 结算轨道)。
> 状态:**草案,待 DAO 敲定数值**。

---

## 0. 与 PGL 的逻辑一致性(这是 PGL 的一个执行实例)

PGL 的核心:**开源免费 + 真实商业行为分账 + 不发可交易代币炒作 + 贡献用不可转让 SBT 记**。aNode 激励必须落在这套逻辑里:

- ✅ **稳态靠真实分润**,不靠币价 —— 对应 PGL "收益对应真实商业行为"。
- ✅ **贡献用不可转让 SBT 累积** —— 对应 PGL "贡献度 SBT 永不可转让"。
- ⚠️ **冷启动用 governance token 一次性激励**(下方 §3 专门说明它**为什么不等于挖矿**、如何与"不发可交易代币炒作"红线自洽)。

---

## 1. 用户为什么愿意跑节点 —— 多重理由叠加(不是单靠钱)

跑节点不能只靠一个理由,要把价值**叠成一摞**,让"相信网络的个体"觉得低风险、多回报、有意义:

| 回报维度 | 是什么 | 现金 / 非现金 |
|---|---|---|
| **基础分润** | 网络真实业务(relay 销售、keeper 保活的 paymaster 毛利、DVT 共签、KMS 验证)的分成 | 现金(随网络量增长) |
| **冷启动 GToken 激励** | 早期业务量低时,用一次性 governance token grant 覆盖硬件成本,把投入风险压到接近 0 | 准现金(治理资产) |
| **声誉 SBT** | 不可转让的贡献声誉:uptime + 服务量 → 解锁更高分润档 + 更高信用额度 + 治理权 | 非现金(长期权益) |
| **费用减免** | 跑硬化节点 → 我们帮他部署"社区大脑"时减免一部分服务费(见 §6) | 等价现金 |
| **可持续前景** | 网络长大 → 他的分润档 + 声誉 + 治理权一起增值;早期参与者占位 | 期权式 |
| **资源支持** | 我们提供一键脚本/远程帮部署/运维支持,把"买→部署→调试→维护"的门槛压到最低 | 降成本 |
| **意义** | 参与数字公共物品、掌握自己社区的身份/验证主权(KMS) | 非经济 |

> 关键叙事:**这不是投机挖矿,是"低风险占位 + 多重回报 + 参与意义"。** 硬件成本由冷启动 GToken 兜底,未来回报随网络真实增长。

## 2. 经济模型 —— 一个个体跑 KMS + DVT,到底划不划算

### 2.1 一次性投入(capex)
| 项 | 估算(¥) |
|---|---|
| 主板(MX93 类) | ~730 |
| 存储 + 外壳 + 电源 + 网络 + 杂项 | ~1100 |
| **合计 capex** | **~1800–1900**(取 ¥1900) |

### 2.2 持续成本(opex,纯 DVT + KMS)
- 电费:MX93 低功耗(~数瓦)→ **~¥1–3/月**,可忽略。
- 带宽:家用已有,边际 ~0。
- gas:**纯 DVT + KMS 几乎不烧 gas**(DVT 共签是链下、KMS 验证是链下;只有额外跑 relay/keeper 才烧 ETH)。
- → **纯节点 opex ≈ ¥3–5/月,基本可忽略。**

### 2.3 时间/精力
- 首次:买 + 部署 + 调试 ≈ 2–4 小时;**我们给一键脚本/远程协助 → 可压到 <1 小时**。
- 日常:~10–30 分钟/月(看告警、偶尔升级)。

### 2.4 回本与回报(关键:让他觉得划算)
opex 近乎为零,所以**划算与否 ≈ 看 capex(¥1900)多久回本 + 之后月入多少**。

| 情景 | 月净分润 | 回本周期 | 回本后年化 |
|---|---|---|---|
| 早期(网络量低) | ¥0–50 | 很久 / 不确定 | 低 → **靠 §3 冷启动 GToken 兜底硬件** |
| 成长期 | ¥150–300 | ~6–12 个月 | ~100–190% |
| 成熟期 | ¥300+ | <6 个月 | 高 |

**怎么让他觉得划算(分三层兜底):**
1. **冷启动 GToken 把硬件风险清零**:早期发等值 ~¥1900 的 GToken grant(覆盖硬件)→ 个体相当于"硬件白拿 + 占位未来分润",**几乎零现金风险**。
2. **稳态基础分润**:随网络量增长,目标把月净收入做到能覆盖时间成本(例如 ¥150–300/月),回本后是被动收入。
3. **非现金叠加**:SBT 声誉(更高分润档/信用/治理)+ 社区大脑费用减免 → 实际回报 > 账面现金。

> 一句话:**对相信网络的个体,这是"¥1900 硬件被 GToken 兜底、近零运维、未来拿真实分润 + 声誉 + 治理"的低风险参与**,而不是赌币价。纯理性投资人也能算得过来——前提是冷启动 grant 真把硬件成本兜住。

## 3. 冷启动:无现金补贴 → governance token 一次性激励(与 PGL 自洽)

社区没有现金,补贴不了。改用 **governance token 一次性激励**桥接早期。**它不是挖矿**,区别在四点:

| | 挖矿(❌ PGL 禁) | 冷启动 GToken grant(✅) |
|---|---|---|
| 发行方式 | 永续按算力/在线 emission,通胀 | **一次性、封顶、里程碑触发**(注册 + 跑满 N 月 + 真实服务过) |
| 依赖 | 币价投机 | 真实参与门槛 |
| 稳态 | 继续靠 emission | **切换到真实分润**(grant 只是早期桥) |
| 性质 | 制造投机资产 | 用既有治理代币做**冷启动对齐**(治理权 + 占位),非炒作 |

→ 即:**早期 = 封顶 GToken grant(代替社区给不起的现金补贴)兜硬件;稳态 = 真实业务分润。** grant 随网络真实收入增长而**退坡到 0**(subsidy→fee 的代币版)。

## 4. 分润比例(草案,弹性区间,沿用 PGL 三角色精神)

PGL 的 Supplier/Wrapper/Seller 弹性区间精神迁移到基础设施:**真正干活的运营方拿大头,协议渠道拿固定小头,区间内由 DAO 议定。**

| 收入来源 | 谁产生 | 运营方分成(建议区间) | 备注 |
|---|---|---|---|
| gasless 销售(GToken/aPoints) | Relay | 运营方覆盖 gas 成本 + 薄利 | 与链上 sale 分账对齐 |
| Paymaster gas 赞助毛利 | Keeper 保活的报价 | keeper 运营方分一部分毛利 | |
| DVT 共签 | 每笔 co-sign | 每笔微费 or 国库/paymaster 补贴 | 量大后转纯分润 |
| KMS 身份验证 | 每次验证/绑定 | 验证服务费分成 | |

数值由 DAO 在区间内敲定(待决策)。

## 5. 贡献度量 + 防作恶

- **可验证贡献**:实际服务的 co-sign / relay / keep / KMS 验证次数 —— 尽量用**链上 tx + 节点 attestation** 度量,不靠自报。
- **防 sybil**:质押(buy governance token → stake,复用 SuperPaymaster v5 角色质押)+ 不可转让 SBT 声誉(刷小号无声誉积累)。
- **防作恶**:BLS 聚合罚没(`executeSlashWithBLS`)+ 信用额度下调。
- **uptime/SLA**:健康探针 + 在线率门槛,达标才计分润档。

## 6. 捆绑"社区大脑"部署服务(额外的真实抓手)

我们另有一个项目:**帮社区建自己的"社区大脑"**(需要更大内存的 Mac / 专用 AI 小主机)。

- **初期我们能做的**:帮他**部署 + 建设**社区大脑,收一部分服务费。
- **激励钩子**:如果他**同时跑一个去中心化硬化节点(KMS + DVT)**,我们**减免一部分社区大脑的服务费**。
- → 把"跑节点"和"他本来就想要的社区大脑"绑在一起,用费用减免做现实抓手(等价现金回报),比单纯发钱更可持续、更对齐。

## 7. 结算实现(复用生态,零/少新合约)

- **SuperPaymaster v5**:`registerRole(ANODE/DVT/KMS)`、信用/债务(`recordDebt`/`repayDebt`)、结算、`executeSlashWithBLS`。
- **OpenPNTs / xPNTs**:计价与支付。
- **SBT**:`safeMint`/`setReputation` 记不可转让声誉。
- 可能需要一个**轻量结算适配**把"服务计数 → 分润"串起来(待评估)。
